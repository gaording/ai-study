# 作业一：痛点挖掘机

## 第一步：观察与记录（30分钟）

### 场景一：每天晚上焦虑AI创业
- **做什么**：学习AI知识
- **为什么烦**：没时间学习，大量时间花费在工作上，AI方向投入不足

### 场景二：周五晚上考虑周末去哪
- **做什么**：思考周末安排
- **为什么烦**：总想既要又要，又想出去玩，又想学习，又想认识新同学

### 场景三：上班期间正常工作
- **做什么**：做一个事，会被各种业务的事情打断
- **为什么烦**：陷入到无限的线程切换中

---

## 第二步：选最值得解决的（1小时）

### 优先级公式：高频率 × 长期存在 × 高时间成本 × 高改变价值

| 场景 | 频率 | 时间成本 | 改变价值 | 问题 |
|------|------|----------|----------|------|
| 场景一：焦虑AI创业 | 每天 | 焦虑消耗精力，但学习时间是0 | 高（如果解决了） | ❌ 这不是"低效反复操作"，而是"资源分配矛盾" |
| 场景二：周末决策瘫痪 | 每周 | 1-2小时纠结 + 后果不佳 | 中 | ❌ 更多是决策心理问题，非流程痛点 |
| 场景三：线程切换被打断 | 每个工作日 | **极高**（上下文切换成本） | **极高** | ✅ **符合"麻烦、低效、反复、不得不做"** |

### ✅ 选定场景：场景三（上班被打断，陷入线程切换）

**理由：**
1. 符合文档描述的特征：麻烦、低效、反复、不得不做
2. 可量化：被打断次数、切换成本、恢复专注的时间
3. 可改进：这是流程/工具问题，不是资源矛盾
4. 高价值：解决了会显著提升工作效率

---

## 第三步：深层分析（2小时）

### ✅ 第一层：现在怎么做（What）

清晰描述现在的做法：

```
我现在的做法：
1. 早上打开电脑，开始处理某个任务（写代码/写文档/思考问题）
2. 花15-30分钟进入专注状态，建立上下文
3. 收到中断：微信消息/飞书@/同事走过来/电话
4. 快速判断：这事儿紧急吗？
5. 看起来紧急 → 立即切换过去处理（3-10分钟）
6. 处理完，想回到之前的任务
7. 不记得刚才写到哪了、思路到哪了
8. 花5-10分钟重新看代码/文档，恢复上下文
9. 刚恢复专注，又被新的中断打断
10. 循环往复，一天重复10-20次

总耗时：
- 每天中断次数：10-20次
- 每次恢复成本：5-10分钟
- 每天浪费在切换上：1-2小时
- 深度工作时间：几乎为0
```

### ✅ 第二层：最想摔键盘的地方（Why烦）

不是"被打断"本身，而是：

```
最想摔键盘的地方：每次恢复都需要重建上下文

为什么烦：
- 每次恢复都需要5-10分钟，一天被切断10-20次
- 累计浪费1-2小时在"切换成本"上
- 深度思考型工作永远无法进入心流状态
- 到了晚上发现"忙了一天，但什么都没做完"
- 同时感到：
  • 内疚（不回消息显得不配合）
  • 焦虑（重要工作做不完）
  • 无力感（无法控制自己的时间）
- 同事/老板觉得你"回复慢"或"不响应"
- 长期导致：只能做浅层工作，无法做需要深度思考的高价值工作
```

### ✅ 第三层：根本原因是什么（Root Cause）

不是工具问题，而是什么问题？

```
根本问题分析：

问：为什么我不想被打断？
答：因为我需要连续的专注时间来完成需要深度思考的工作

问：为什么不能拒绝打断？
答：因为工作文化要求"及时响应"，不敢"失联"

问：如果有一个"勿扰模式App"，能解决吗？
答：不能。因为：
- 开了勿扰，会错过真正紧急的事情
- 关了勿扰，又会被打断
- 问题不在于"要不要勿扰"，而在于"如何判断什么该打断"

问：那真实问题是什么？
答：
1. 信息流是"平等"的（所有消息都弹窗），没有优先级过滤
2. 别人不知道我"正在专注"，无法判断"现在是否适合打断我"
3. 我不知道消息的"紧急程度"，不敢不立即查看
4. 没有"缓冲机制"：要么完全拒收，要么全盘接收

根本原因：不是"我容易被干扰"
而是"工作环境的实时通讯机制与深度工作需求不匹配"

具体来说：
- 缺乏"可中断性"的显性标识
- 缺乏"消息优先级"的自动分流
- 缺乏"异步沟通"的文化支持
```

## 第四步：用户识别（30分钟）

### ✅ 这个问题的受害者是谁？

```
用户是谁：知识工作者中的"深度工作者"

具体群体：
- 软件工程师/程序员
- 产品经理（需要写PRD、做深度分析）
- 数据分析师/算法工程师
- 研究员/分析师
- 创业者（在职创业的斜杠青年）

特征：
- 工作内容需要"连续的专注时间"（30分钟以上）
- 需要处理大量"上下文依赖"的任务（写代码、写文档、思考方案）
- 同时处于"高协作环境"（需要与团队频繁沟通）
- 使用多种通讯工具：微信、飞书/钉钉、邮件、电话、面对面

为什么有这个痛点：
1. 工作性质决定：需要深度思考，但工作环境要求高度协作
2. 工具冲突：Slack/飞书/微信这类工具的设计就是"实时打断"
3. 文化矛盾：公司要求"快速响应"，但深度工作需要"慢响应"
4. 缺乏工具支持：现有工具没有"可中断性管理"功能
5. 个人无力感：一个人无法改变整个团队的沟通习惯

规模：
- 我认识的至少10个人有这样的困扰
- 程序员群体中几乎普遍存在
- 远程办公后更严重（因为无法用"戴耳机"来物理隔绝）
```

## 第五步：提炼用户真实需求（1小时，最关键）

### ❌ 常见错误（工具思维）

```
❌ "我需要一个勿扰模式App"
❌ "我需要一个智能拦截软件"
❌ "我需要一个消息过滤工具"
❌ "我需要一个番茄钟定时器"
（这些都还在"工具思维"里，只是在现有工具上打补丁）
```

### ✅ 剥离工具形状，找到真实需求

```
第一步：剥离"工具形状"

不问"我需要什么App"，而是问"我真正想要什么状态"

我想要的状态是：
✓ 当我进入深度工作时，别人能知道"现在不适合打断我"
✓ 真正紧急的事情，能穿透进来找到我
✓ 不紧急的事情，能被自动"暂存"，等我方便时批量处理
✓ 我不需要"判断每条消息是否紧急"，系统帮我分流
✓ 我不需要"因为不回消息"而感到内疚
✓ 我能主动控制自己的"可中断性"

第二步：抽象成本质需求

本质需求不是"管理消息"，而是：

"管理我的'可中断性状态'，并让这个状态对他人可见"

具体拆解：
1. 状态管理：我能声明"我现在进入深度工作模式"
2. 状态可见：协作者能看到我的状态，知道"现在不适合打断"
3. 智能分流：系统自动区分"紧急"vs"可延迟"的消息
4. 优雅降级：非紧急消息被"暂存"，不被丢失，而是延后处理
5. 主动恢复：我能主动查看"暂存区"，批量处理
6. 紧急穿透：真正紧急的事能穿透（系统判断+我设置白名单）

第三步：用一句话描述真实需求

"我需要一套'可中断性管理机制'，让我的专注时间得到保护，同时不遗漏真正重要的事情"

这不是一个"勿扰App"，而是：
- 一种新的工作协议/机制
- 一种"异步优先"的沟通范式
- 一种让"深度工作"和"高效协作"并存的解决方案
```
